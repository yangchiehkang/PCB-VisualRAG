论文审阅报告：PCB 工程文档的预算化多向量视觉检索与 Visual RAG
整体来看，这是一篇结构完整、实验扎实、叙事清晰的本科毕业论文 👏。研究问题聚焦、方法链条递进、结果与讨论自洽，尤其在"BM25 强基线 + 视觉检索作为成本可控补充分支"这一定位上把握得相当成熟。下面按 结构性问题 → 实验与论证 → 表达与格式 → 细节勘误 四层给出可操作的修改建议。

🧭 一、整体结构与定位（优先级最高）
✅ 做得好的地方
摘要、引言、相关工作、方法、实验、讨论、局限、结论的链条完整。
明确交代了与 Proposal / Midterm 的关系，体现了研究演进。
中心问题（"Can visual multi-vector retrieval be made accurate, interpretable, and cost-efficient...?"）贯穿全文。
⚠️ 建议改进
核心矛盾需要在引言中更早"亮牌"

你的主要发现其实是一个略反直觉的结论："视觉多向量检索没赢过 BM25，但预算化视觉分支显著优于原始 Full MV。" 建议在 Introduction 末尾的 bullet 列表之前，用一句话把这个张力点出来，例如：

"This study finds that visual multi-vector retrieval does not surpass OCR BM25 on this corpus, but a budgeted and quantized visual branch substantially improves over its own Full MV baseline — reframing visual retrieval as a cost-controlled complement rather than a replacement."

第 1.1 节与摘要内容重复度偏高

摘要、1.1、第 10 节结论里同样的数字（0.8833 / 0.5241 / 0.1249 / 0.1296）出现了 3–4 次。建议：

摘要保留全部关键数字；
1.1 只保留"研究阶段演进 + 一句话总结发现"；
结论部分换一种表述（例如用"相对提升百分比"而非重复绝对值）。
缺少一张总览性的 Pipeline 图

全文方法链条很长（OCR → SV → Full MV → C2F → Token Budget → PQ → Evidence → VRAG），但没有一张总流程图。强烈建议在第 4 节开头加一张 System Overview Figure，读者会非常感激。

🔍 二、实验与论证层面的疑点
这一部分是审阅人（或答辩老师）最可能追问的，请认真核对。

1. Recall@10 在不同表格中数值不一致 ⚠️ 重要
Table 3（Baseline）：Full Multi-vector Visual 的 Recall@10 = 0.1333
Table 5（Token Budget）：C2F N10 M49（等价于 full token）Recall@10 = 0.2667
Table 4：BM25 + Full MV 的 Recall@10 = 0.8833
读者会困惑："Full MV 到底是 0.1333 还是 0.2667？" 实际上这是三种不同设定（独立全库 MV / BM25 候选 + MV 重排 / C2F 内部全 token），但你没在表注里说清楚。建议：

在 Table 5 加脚注："C2F N10 M49 reranks BM25 top-10 candidates using all 49 tokens; this differs from the standalone Full MV in Table 3, which ranks the entire corpus."
或者把三种"Full MV"分别命名为 Full MV (standalone)、Full MV (rerank)、C2F-Full，避免歧义。
2. Table 4 的 "Budget / Unit" 列含义不明
列里出现 3030 / 10 / 20 / 301 这些数字，正文只说"effective retrieval budget or index unit"，但读者根本看不懂单位是什么。建议：

改成两列：Candidate N 和 Rerank Cost；
或者明确写"queries × pages scored"等具体含义。
3. Token Budget 结果的解释存在小漏洞
"M = 8 retains only 16.33% of visual tokens but achieves the best nDCG@10"

但表里 M=8、16、24 的 Recall@10 都是 0.2667，且 nDCG@10 差距很小（0.1249 vs 0.1242 vs 0.1145）。在 30 个查询、10 个 evidence 子集的体量下，这些差异很可能不显著。建议：

补一句："Differences across M ∈ {8,16,24} are within the noise range of a 30-query benchmark; the key takeaway is that aggressive token reduction does not degrade retrieval."
这反而比"M=8 最好"更稳妥、更可信。
4. QA-level Occlusion 全是 1.0000 的结果需要更强解释
Table 12 三种条件（Original / Gold Masked / Random Masked）准确率都是 1.0000 / 1.0000 / 1.0000，Delta 全为 0。你在 7.5 节解释为"VLM 利用了页码、标题等线索"。这个解释合理，但：

结果太干净反而显得实验设计有问题。建议明确写："This null result is itself a methodological finding: page-identification questions are too weak a probe for region-level causal analysis; future work should adopt content-specific questions (e.g., '请读出此区域中的电阻值')."
否则审阅人会质疑"那这个实验做了等于没做"。
5. Evidence Hit@10 = 1.0000 (BM25) 与 Recall@10 = 0.8833 不一致
Table 3 中 BM25 的 Recall@10 = 0.8833（30 query），但 Table 7 的 Evidence Hit@10 = 1.0000（10 query 子集）。这是因为 evidence 子集是更容易的 10 个 query 吗？需要在 3.3 节明确说明子集的构造方式（是随机抽样还是挑选了有清晰 region 的样本？），否则会被质疑"子集偏置"。
